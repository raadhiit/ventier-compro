# 25. Definition of Done

A feature is complete only when:

- The business requirement is satisfied.
- The UI is responsive.
- Validation is implemented.
- Admin authorization is verified where applicable.
- Empty and error states are handled.
- No obvious excessive queries remain.
- Media is optimized.
- Important tests exist, or the exception is documented.
- No placeholder or debug output remains.
- Administrators can manage the intended content within the approved scope.
- The feature has been tested in an environment reasonably close to production.
